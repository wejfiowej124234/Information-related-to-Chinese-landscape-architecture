## 1 资质获取与投标资料路线图（总览）

```mermaid
flowchart TD
  A[公司主体建立] --> B[营业执照]
  B --> C[基础合规与资料底座\n章/税务/社保/制度/台账]

  C --> D[住建体系：施工资质]
  D --> E[住建体系：安全生产许可证（安许）]
  E --> F[工程类项目：投标/施工准入]

  B --> G[银行账户材料\n（投标/合同/收付款归档）]
  G --> F

  C --> H[投标常见加分/准入材料\nISO三体系 / AAA信用 / 能力评价 / 专项服务评价]
  H --> F

  C --> I[专项业务合规材料\n（渣土/消纳/运输/去向台账等，属地口径）]
  I --> F
```

## 2 什么项目 → 需要什么资质（对齐图）

## 3 项目类型 → 常见门槛/加分（决策树）

```mermaid
flowchart LR
  P[项目需求出现] --> Q{项目采购属性？}

  Q -->|工程类| E1[优先看：施工资质 + 安许\n（以招标文件/属地口径为准）]
  E1 --> E2[再看：人员/业绩/在线核验（四库等）]
  E2 --> E3[加分：ISO/信用/能力评价（如计分）]

  Q -->|服务类| S1[优先看：主体合规/服务方案/响应能力\n（可能要求入库/供应商准入）]
  S1 --> S2[如含工程量：可能追加施工资质+安许]
  S2 --> S3[加分：ISO/信用/案例与台账]

  Q -->|货物类| G1[优先看：营业执照/供货能力/检测或质保\n（按招标要求）]
  G1 --> G2[加分：ISO/信用/供货业绩]
```

## 5 项目级别（门槛强度）→ 资质组合（架构图）

> 说明：这里的“级别”按招标文件的门槛强度理解（门槛/加分/入库），而非固定金额或固定等级；最终以招标文件/属地口径为准。

```mermaid
flowchart TB
  A0[所有项目通用底座] --> A1[营业执照/主体合规]
  A1 --> A2[投标文件规范\n（格式/盖章/清晰度/一致性）]

  A2 --> P{项目属性}
  P -->|工程施工类| E0[工程施工类：门槛更强]
  P -->|服务类（养护/运维/咨询）| S0[服务类：看准入规则]
  P -->|货物类（苗木/设备/材料）| G0[货物类：看供货与质保]

  E0 --> E1[门槛项（常见）\n施工资质（住建）+ 安许]
  E1 --> E2[配套项（常见）\n人员/业绩/在线可核验]
  E2 --> E3[加分项（如计分）\nISO/信用/能力评价]

  S0 --> S1[门槛项（常见）\n主体合规 + 服务方案/响应能力]
  S1 --> S2[如含工程量/施工内容\n可能追加：施工资质 + 安许]
  S2 --> S3[加分项（如计分）\nISO/信用/案例与台账]

  G0 --> G1[门槛项（常见）\n主体合规 + 供货能力/检测/质保]
  G1 --> G2[加分项（如计分）\nISO/信用/供货业绩]

  E3 --> OUT[输出：资质对齐清单 + 材料打包表]
  S3 --> OUT
  G2 --> OUT
```
